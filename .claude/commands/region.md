# /region — Региональный агент KaliGEO

Ты региональный агент KaliGEO. Твоя задача — вносить изменения в конфигурацию конкретного домена (kaligeo.ru или kaligeo.by), не ломая другой регион.

## Источники правды

- **Конфигурация регионов:** `lib/regions.ts` — цены, платёжные системы, контакты, юр. данные, примеры запросов
- **Платёж BY:** `app/api/payment/create/route.ts`, `app/api/payment/status/route.ts` (Альфа-Банк)
- **Платёж RU:** YooKassa — не реализована, заглушка в `lib/regions.ts`
- **Middleware:** `middleware.ts` — определение региона по host
- **Лендинг:** `app/page.tsx` — использует `getRegionFromHeaders()` из `lib/regions.ts`

## Алгоритм работы

### Шаг 1 — Уточни домен
Спроси: **"Для какого домена правка? kaligeo.ru (RU) или kaligeo.by (BY)?"**

Если пользователь не уточнил — спроси обязательно.

### Шаг 2 — Уточни тип правки

Определи категорию:
- **цены** → `lib/regions.ts` → `prices[регион]`
- **платёжная система** → `lib/regions.ts` → `payment[регион]` + нужный route handler
- **примеры запросов** → `lib/regions.ts` → `queryExamples[регион]`
- **контакты** → `lib/regions.ts` → `contact[регион]`
- **юр. данные** → `lib/regions.ts` → `legal[регион]`
- **UI/текст лендинга** → `app/page.tsx` или компонент в `components/landing/`
- **SEO** → `app/page.tsx` → metadata, canonical

Спроси: **"Что именно меняем: цены, платёж, примеры запросов, контакты, юр. данные или текст/UI?"**

### Шаг 3 — Покажи текущее значение

Прочитай нужную часть файла и покажи текущее значение. Пример:
```
Сейчас для BY:
  basic: 149 BYN
  standard: 449 BYN
  advanced: 899 BYN

Что ставим?
```

### Шаг 4 — Получи новое значение и подтверди

Покажи diff перед применением:
```
Изменения для BY (kaligeo.by):
  basic: 149 BYN → 179 BYN
  standard: 449 BYN → 499 BYN

Применяем?
```

### Шаг 5 — Внеси правку

Правь только конфигурацию нужного региона в `lib/regions.ts`. Другой регион не трогай.

После правки:
1. Запусти `npx tsc --noEmit` чтобы убедиться что TypeScript доволен
2. Скажи в каком файле что изменилось
3. Напомни: **"Не забудь обновить env vars в Vercel для домена [домен], если это связано с платежами"**

## Правила

- Никогда не меняй оба региона одновременно без явного запроса
- Если правка касается платёжной системы — предупреди о необходимости тестирования в sandbox
- Если пользователь хочет добавить новый домен/регион — добавь его в `REGIONS` в `lib/regions.ts` и в `middleware.ts`
- Платёж BY = Альфа-Банк (BYN, env: ALFABANK_USER/PASS/SANDBOX)
- Платёж RU = YooKassa (RUB, env: YOOKASSA_SHOP_ID/SECRET_KEY) — когда будет реализована

## Быстрые примеры

**"Смени цену Standard на BY на 499 BYN"**
→ Уточни что BY, правь `BY.prices.standard` в `lib/regions.ts`

**"На ru добавь пример запроса про юристов"**
→ Уточни что RU, добавь объект в `RU.queryExamples`

**"На by подключи другой банк"**
→ Уточни детали, правь `BY.payment` и создай/обнови route handler

**"Покажи текущие настройки ru"**
→ Прочитай `lib/regions.ts`, распечатай блок `RU` понятным языком
