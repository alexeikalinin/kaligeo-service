# Skill: recommend-tier

Помощник по выбору тарифа KaliGEO для потенциального клиента.

## Когда использовать

Когда нужно определить оптимальный тариф (Basic/Standard/Advanced) перед продажей аудита.

## Алгоритм

Задай клиенту (или проанализируй по описанию бизнеса) следующие вопросы:

### 1. Как люди ищут этот бизнес?

**По названию бренда** → сигнал к Basic/Standard
- Пример: "KFC Минск", "Wildberries скидки", "Notion review"

**По типу услуги/продукта** → сигнал к Standard/Advanced
- Пример: "гинеколог Минск", "ремонт iPhone Москва", "юрист по недвижимости"
- Если таких запросов много и они разные — нужно больше запросов (Standard или Advanced)

### 2. Количество услуг / продуктов

- 1–3 услуги/продукта → Basic достаточно
- 4–10 услуг → Standard (нужно покрыть все направления запросами)
- 10+ услуг / каталог / клиника с отделениями → Advanced (150 запросов)

### 3. Конкурентная ситуация

- Мало конкурентов, нишевой рынок → Basic
- Средняя конкуренция, есть 3–5 прямых конкурентов → Standard
- Высокая конкуренция, лидеры рынка → Advanced (нужна матрица конкурентов + агенты)

### 4. Цель аудита

- "Просто посмотреть как дела" → Basic
- "Нужен план действий + PDF для команды" → Standard
- "Хотим стратегию + понять пробелы + исправить сайт" → Advanced

### 5. Горизонт работы

- Разовая проверка → Basic
- Квартальный мониторинг → Standard (есть сравнение с прошлым аудитом)
- Постоянная работа над видимостью → Advanced (неограниченный чат, все агенты)

## Таблица принятия решений

| Тип бизнеса | Типичный выбор | Почему |
|---|---|---|
| Cafes, рестораны (известное название) | Basic | Ищут по бренду, 3 платформы достаточно |
| SaaS-продукт / стартап | Basic → Standard | Один продукт, но нужна платформа Perplexity и Claude |
| Медцентр / клиника (много услуг) | Standard → Advanced | Ищут "терапевт Минск", "УЗИ Минск" — нужно 30–50 разных запросов |
| Юридическая фирма (несколько практик) | Standard | Ищут "юрист по ДТП", "развод юрист" — запросы по услугам |
| E-commerce / маркетплейс | Advanced | Категории, бренды, конкуренты — нужна матрица + агенты |
| Образование (курсы, школа) | Standard | Ищут по предмету + городу или онлайн |
| Агентство недвижимости | Standard → Advanced | Ищут по типу сделки и локации |
| B2B сервис / завод / производство | Basic → Standard | Узкая ниша, но важно покрыть Perplexity и DeepSeek |
| Гостиница / отель | Basic | Ищут по названию или геолокации, не по категории |
| Медиа / блог / личный бренд | Basic | Один автор/бренд |

## Ключевой вопрос для медицины и услуг

> Скажите, когда ваш типичный пациент/клиент ищет помощь — он вводит название вашей клиники или конкретную услугу (например, "удаление зуба Минск")?

Если **услугу** → нужно минимум Standard, скорее всего Advanced.

Количество запросов в пайплайне напрямую = количество «ниш», которые мы проверяем. 15 запросов (Basic) покроют 1–2 услуги. 30 (Standard) — 5–8 услуг. 50 (Advanced) — 10–15 услуг с вариациями.

```
- [конкретно чего не хватит]
```
