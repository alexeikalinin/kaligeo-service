# YandexGPT API Key

tags: #api-keys #setup #yandex #yandexgpt

**Нужен для:** YandexGPT + Алиса — оба клиента используют эти же ключи (Basic+)  
**Бесплатно:** 1 000 000 токенов/день в тестовом режиме  
**Нужно:** Яндекс аккаунт + платёжный аккаунт (карта не снимается при тестировании)

---


## Шаг 3 — Создать folder (каталог)

Каталог создаётся через меню облака, **не через "Создать ресурс"**.

**Способ 1** (через облако):
```
Верхняя панель → нажать "..." рядом с названием облака (cloud-alexeikalinin1)
→ "Создать каталог"
→ Имя: "kaligeo"
→ Описание: (пусто)
→ Метки: (пусто)
→ "Создать сеть по умолчанию" ← снять галочку (не нужна для API)
→ Создать
```

**Способ 2** (через текущий каталог):
```
Верхняя панель → нажать стрелку "∨" рядом с именем текущего каталога (default)
→ "Создать каталог"
→ Имя: "kaligeo"
→ Описание: (пусто)
→ Метки: (пусто)
→ "Создать сеть по умолчанию" ← снять галочку (не нужна для API)
→ Создать
```

После создания — **скопировать ID каталога**. Самый простой способ:

```
Верхняя панель → нажать "..." рядом с именем каталога "kaligeo"
→ "Копировать ID"
→ ID скопирован в буфер — это YANDEX_FOLDER_ID
```

Альтернативно — ID виден прямо в шапке рядом с именем каталога:
```
kaligeo   b1g0224vthr9vu8v3vga  ← это YANDEX_FOLDER_ID
```

## Шаг 4 — Создать сервисный аккаунт

```
Верхняя панель → нажать "..." рядом с именем каталога "kaligeo"
→ "Создать сервисный аккаунт"
→ Имя: "kaligeo-sa"
→ Описание: (пусто)
→ Роли → "+ Добавить роль" → в поиске набрать "ai.languageModels.user" → выбрать
→ Нажать "Создать"
```

> ⚠️ Меню "..." в шапке открывает опции для **текущего каталога**. Убедиться, что активен **kaligeo**, а не default.

После нажатия "Создать" откроется страница нового сервисного аккаунта — перейти сразу к Шагу 5.

---

## Шаг 5 — Создать API ключ

Если страница сервисного аккаунта не открылась автоматически — найти его:
```
Левое меню → "Identity and Access Management"
→ "Сервисные аккаунты"
→ Нажать на "kaligeo-sa" в списке
```

На странице сервисного аккаунта нажать **"+ Создать новый ключ"** (кнопка в правом верхнем углу).

Появится меню из трёх вариантов — выбрать **"Создать API-ключ"**:
```
☐ Создать статический ключ доступа  ← не нужен (для S3/хранилищ)
✅ Создать API-ключ                  ← нужен этот (упрощённая аутентификация)
☐ Создать авторизованный ключ       ← не нужен (для IAM-токенов)
```

В форме создания:
```
→ Описание: "kaligeo-production"
→ Область действия → выбрать "yc.ai.languageModels.execute"
  (можно набрать в поиске: "language")
→ Срок действия: оставить пустым (бессрочный)
→ Нажать "Создать"
```

⚠️ Ключ показывается **ОДИН РАЗ**. Сразу скопировать:

```
Секретный ключ: AQVN...  ← это YANDEXGPT_API_KEY — скопировать!
Идентификатор: (не нужен)
```

---

## Шаг 6 — Добавить в .env.local

```bash
YANDEXGPT_API_KEY=AQVN...    # API ключ из шага 5
YANDEX_FOLDER_ID=b1g...      # ID каталога из шага 3
```

---

## Проверка

Тест через curl:
```bash
curl -X POST \
  -H "Authorization: Api-Key $YANDEXGPT_API_KEY" \
  -H "x-folder-id: $YANDEX_FOLDER_ID" \
  -H "Content-Type: application/json" \
  -d '{"modelUri":"gpt://b1g.../yandexgpt/latest","completionOptions":{"stream":false,"temperature":0.3,"maxTokens":"100"},"messages":[{"role":"user","text":"Привет!"}]}' \
  https://llm.api.cloud.yandex.net/foundationModels/v1/completion
```

Ожидаемый ответ: JSON с `result.alternatives[0].message.text`
